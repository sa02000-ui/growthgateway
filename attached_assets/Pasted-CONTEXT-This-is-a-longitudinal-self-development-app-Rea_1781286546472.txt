CONTEXT
This is a longitudinal self-development app (React + Vite client, Express server, Supabase for 
auth/data + a separate Replit Postgres via DATABASE_URL for profiles/life-events). Users take 
psychometric assessments over time, log life events in their profile, and collect peer feedback to 
compare self-perception vs how others see them. A prior pass added requireAuth/getUserId 
(server/auth.ts) and rate limiting (server/rate-limit.ts), removed the Groups feature, hardened 
types, and locked down RLS — DO NOT regress any of that. Every protected route must keep using 
requireAuth and the token's user id, never a userId from the URL/body.

Implement the following in priority order. Create DB migrations as needed. After each phase, run 
`npm run check` (tsc) and confirm it passes.

==========================================================
PHASE 1 — Distinguish stable TRAITS from changeable STATES (most important)
==========================================================
Problem: the app trends all tests the same way, but Big Five, Dark Triad, RIASEC and the ICAR 
cognitive test are stable/fixed — retaking them monthly shows noise + practice effects, not growth. 
Only wellbeing/state measures (PSS-10, SWLS, BRS, Flourishing) meaningfully change short-term.

1. Add a `measurementClass: 'trait' | 'state'` field and a `retestIntervalDays` number to each 
   assessment's seed metadata (shared/assessments/*.ts and the library builder in server/routes.ts).
   - trait (revisit ~yearly, retestIntervalDays ~365): IPIP Big Five, PVQ, SD3, RIASEC, ICAR.
   - state (revisit anytime, retestIntervalDays ~7–30): PSS-10, SWLS, BRS, Flourishing, and all 
     new state tests added later.
2. ICAR cognitive test: mark as ONE-TIME. After a user has one result, disable retaking and show 
   "Cognitive ability is stable; repeat testing mainly measures item memory." Remove ICAR from any 
   time-series/trend view.
3. Reliable Change Index: where an assessment has a Cronbach's alpha and an SD, compute 
   RCI = 1.96 * SD * sqrt(2) * sqrt(1 - alpha). Only label a change between two results as 
   "meaningful" when |change| exceeds the RCI. Add a shared util (e.g. shared/reliable-change.ts) and 
   use it in client comparisons (replace the raw point-difference logic in 
   client/src/components/dashboard/home-tab.tsx calculateComparison).
4. On all trend lines, render a shaded confidence band (±RCI or ±1 SEM) so users don't 
   over-interpret noise.
